Queenly Take-Home Assignment

Background Info: Within this folder, you will see a csv file named "wardrobe_items_downloaded_data.csv". This is a csv representation of approximately 8,000 dresses from our database. The fields values are:

  • wardrobe_id: The id of the dress item
  • wardrobe_size: The dress size, ex. "4" means a dress size 4
  • wardrobe_color: The color of the dress
  • wardrobe_style: The style/shape of the dress
  • wardrobe_price: The price of the dress, in $
  • wardrobe_created_at_ms: The timestamp of when the dress was listed for sale, in UNIX
  • wardrobe_designer: The fashion designer/brand label of the dress
  • wardrobe_photo_signature: The photo signature of the dress, to display the URL. ex: if the photo signature value is "wardrobe-1610511485308", the url of the photo will be: https://qny.queenly.com/wardrobe_grid/wardrobe-1610511485308.jpg

      • Instructions

      Using the provided data source of dresses, update the python file "run.py" that generates 100 recommended dresses for the users represented in "trisha.json" and "emily.json"

      These 100 dresses should be ranked from most to least similar to their preferences, and show some similarity to the dresses in each users "last_saved_dresses" field.

      The output should be 4 files:

      • trisha.txt <= a list of wardrobe ids, ordered from most to least similar to their preferences. There is an example in "example_output.txt"
      • trisha.html <= a visual representation of trisha.txt. There is an example in "example_output.html"
      • emily.txt <= a list of wardrobe ids, ordered from most to least similar to their preferences. There is an example in "example_output.txt"
      • emily.html <= a visual representation of trisha.txt. There is an example in "example_output.html"

      You may reuse the html and css used in example_output.html to represent your data
